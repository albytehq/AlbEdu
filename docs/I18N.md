# I18N — Internationalization System

> AlbEdu v1.0.0 supports 5 languages: Indonesia (default), English, Russia, Spanyol, Mandarin.
> Implementation: Phase 8.

---

## 1. Supported Locales

| Code | Language | Native Name | Direction | Status |
|---|---|---|---|---|
| `id` | Indonesia | Bahasa Indonesia | LTR | Default, complete |
| `en` | English | English | LTR | Complete |
| `ru` | Russia | Русский | LTR | Complete |
| `es` | Spanyol | Español | LTR | Complete |
| `zh` | Mandarin | 中文 | LTR | Complete |

**Arabic (RTL)** deferred to Phase 9 (complex RTL layout changes).

---

## 2. Architecture

```
src/i18n/
├── index.js              # i18n engine (load locale, t(), switchLocale())
├── detector.js           # Auto-detect browser language
└── locales/
    ├── id.json           # Indonesia (default, 100% complete)
    ├── en.json           # English
    ├── ru.json           # Russia
    ├── es.json           # Spanyol
    └── zh.json           # Mandarin
```

### 2.1 i18n Engine (`src/i18n/index.js`)

**Public API:**
```js
window.i18n.t(key, params?)           // Translate key with optional interpolation
window.i18n.switchLocale(locale)      // Switch active locale
window.i18n.getCurrentLocale()        // Get current locale code
window.i18n.getSupportedLocales()     // Get list of supported locales
window.i18n.onLocaleChange(callback)  // Subscribe to locale changes
```

**Usage in JS:**
```js
const t = window.i18n.t;
document.querySelector('#page-title').textContent = t('nav.create_assessment');
// With interpolation:
document.querySelector('#timer').textContent = t('take.time_remaining', { minutes: 15 });
// "Sisa Waktu: 15 menit" (id) / "Time Remaining: 15 minutes" (en)
```

**Usage in HTML:**
```html
<span data-i18n="nav.create_assessment">Buat Asesmen</span>
<span data-i18n-attr="placeholder:take.search_placeholder">Cari...</span>
```

### 2.2 Auto-Detection (`src/i18n/detector.js`)

**Detection order:**
1. `localStorage.getItem('albedu_locale')` — user's manual choice
2. `navigator.language` — browser language (e.g. `id-ID`, `en-US`, `ru-RU`)
3. Default: `id` (Indonesia)

**Mapping:**
- `id`, `id-ID` → `id`
- `en`, `en-US`, `en-GB` → `en`
- `ru`, `ru-RU` → `ru`
- `es`, `es-ES`, `es-MX` → `es`
- `zh`, `zh-CN`, `zh-TW` → `zh`
- Others → fallback to `id`

## 4. Features

### 4.1 Interpolation

```js
t('take.time_remaining', { minutes: 15 })
// id: "Sisa Waktu: 15 menit"
// en: "Time Remaining: 15 minutes"
```

### 4.2 Pluralization

```json
{
  "questions_count": {
    "zero": "Tidak ada soal",
    "one": "1 soal",
    "other": "{{count}} soal"
  }
}
```

```js
t('questions_count', { count: 0 })  // "Tidak ada soal"
t('questions_count', { count: 1 })  // "1 soal"
t('questions_count', { count: 5 })  // "5 soal"
```

### 4.3 Date/Number Formatting

Uses native `Intl.DateTimeFormat` + `Intl.NumberFormat`:

```js
const date = new Date('2026-07-01');
window.i18n.formatDate(date);  // id: "1 Juli 2026", en: "July 1, 2026"
window.i18n.formatNumber(1234.56);  // id: "1.234,56", en: "1,234.56"
```

### 4.4 Fallback

If key missing in active locale, fallback to Indonesia (`id`). If missing in Indonesia too, return key itself.

### 4.5 RTL Support (Future — Phase 9)

Arabic (`ar`) locale will require:
- `dir="rtl"` on `<html>`
- CSS mirror (left ↔ right swap)
- Icon direction flip (arrows, back button)
- Logical properties (`margin-inline-start` instead of `margin-left`)

## 5. Adding a New Locale

### Step 1: Create locale file

Copy `id.json` → `xx.json` (where `xx` is ISO 639-1 code), translate all values.

### Step 2: Register in `index.js`

```js
const SUPPORTED_LOCALES = {
  id: { name: 'Bahasa Indonesia', native: 'Bahasa Indonesia', dir: 'ltr' },
  en: { name: 'English', native: 'English', dir: 'ltr' },
  // ... add new locale:
  ar: { name: 'Arabic', native: 'العربية', dir: 'rtl' },
};
```

### Step 3: Update detector.js

Add mapping in `mapBrowserLocale()`:
```js
const LOCALE_MAP = {
  'ar': 'ar', 'ar-SA': 'ar', 'ar-EG': 'ar',
  // ...
};
```

### Step 4: Update env

```env
ALBEDU_SUPPORTED_LOCALES=id,en,ru,es,zh,ar
```

### Step 5: Test

- Switch to new locale via UI
- Verify all text translated
- Verify date/number formatting
- Verify RTL layout (if applicable)

## 7. Translation Workflow

### 7.1 Initial Translation

1. Complete `id.json` (default, by developer)
2. Translate to other 4 languages:
   - Option A: Native speaker translator
   - Option B: AI translation (DeepL / Google Translate) + human review
   - Option C: Community contribution (future)

### 7.2 Ongoing Maintenance

- When new feature added, add keys to `id.json` first
- Translation tool flags missing keys in other locales
- Fallback to `id` ensures no blank UI

### 7.3 Translation Quality

- Use formal/professional tone (not casual)
- Consistent terminology (e.g. "asesmen" not "ujian" in Indonesian)
- Avoid idioms that don't translate
- Test with native speakers if possible

---

## 8. Performance

- Locale files are JSON, ~10-50KB each
- Loaded lazily (only active locale on initial load)
- Other locales loaded on-demand when user switches
- Cached in memory after first load
- Service Worker cache (Phase 8 PWA) — instant locale switch on repeat visits

---

## 9. Compliance Note

- UU PDP (Indonesia): consent popup must be in Indonesian (default locale)
- User can switch to preferred language, but legal documents (Privacy Policy, Terms of Service) are authoritative in Indonesian
